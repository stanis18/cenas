1. INTRODUÇÃO                                                                                                                                              

 

Este documento apresenta as funcionalidades do Sistema CenAS – Cenários de Aprendizagem: Aprendendo a Aprender, o qual deverá ser utilizado como guia de utilização do sistema pelos alunos do curso de medicina da Universidade Federal de Pernambuco.

 

1.1 ESCLARECIMENTOS ADICIONAIS

 

A utilização da opção “Ajuda” do sistema é recomendada para obter as informações de uso e preenchimento dos campos das telas do sistema CenAS. Para ter acesso a essa funcionalidade selecione o botão “Ajuda”.

 

 

2. ACESSO AO SISTEMA

 

Esta tela permite o acesso ao Sistema CenAS, através de uma validação de Login e Senha.

 

 

 

 

2.1 Tipos de usuário

 

2.1.1 Usuários e privilégios

 

O sistema possui diferentes tipos de usuários com perfis de acesso apropriados:

 

·         Acesso livre: este tipo de acesso permite a utilização da funcionalidade de consulta dos dados dos problemas. Poderá ser utilizada por qualquer usuário sem necessidade de informar o login/senha.

 

·         Acesso para controle de disciplinas e seus respectivos problemas: permite aos professores e administradores fazerem a manutenção das informações das disciplinas às quais se encontram vinculados. Podem criar, editar e remover problemas, além de abrir (disponibilizar um problema para edição) ou fechar um problema (impossibilitando a sua para edição).

 

·         Acesso para atualização de problemas: permite aos estudantes que compõem as equipes atualizarem os dados dos problemas em desenvolvimento. Será disponibilizado acessos individuais para cada integrante de uma equipe.

 

·         Acesso para acompanhamento de problemas: permite aos tutores incluírem as informações iniciais para estudo de um determinado tema/problema e acompanharem o andamento dos problemas aos quais estão vinculados. Permite ainda fazer a avaliação dos alunos das equipes que acompanham.

 

·         Cadastro de alunos e manutenção do site: permite ao administrador cadastrar os alunos do período, além de outras opções de administração.

 

2.1.2 Login/Senha

 

O acesso ao sistema é realizado através do Login/Senha do usuário. Após o procedimento de Login, os dados dos problemas correspondentes poderão ser atualizados pelo usuário. Apenas os problemas correspondentes ao Login do usuário serão disponibilizados para atualização. Os demais problemas em desenvolvimento poderão ser consultados através do acesso livre.

 

            2.1.2.1 Alterar Login/Senha

Cada usuário receberá um login de acesso com senha temporária que poderá ser modificada para uma senha pessoal de mais fácil memorização.

 

2.1.3 Cadastro de usuários no sistema

 

Esta funcionalidade oferece ao usuário administrador/professor um cadastro no sistema dependendo do seu nível de acesso. O usuário pode ter privilégios apenas de consulta ou atualizações de dados do sistema. O cadastro de usuários será realizado pelo administrador/professor do sistema. Como se pode ver, o sistema apresenta quatro níveis de acesso diferenciados, sendo eles:

 

·         Professor

 

·         Tutor

 

·         Aluno

 

·         Administrador

4. Atualização e Acompanhamento de Problemas

 

O sistema CenAS foi desenhado para permitir ao estudante do Curso de Medicina registrar o desenvolvimento de um problema em estudo em suas várias fases:

·         Planejamento (realizado pelo tutor):

o   Palavras-chave

o   Cenário

o   Objetivos

o   Justificativa

o   Plano de Desenvolvimento

o   Referências Iniciais

·         Apoio (realizado pelos estudantes da equipe):

o   Glossário

o   Referências Bibliográficas

o   Material Pesquisado

o   Material Gráfico

·         Desenvolvimento (realizado pelos estudantes da equipe):

o   Relatório das Atividades

o   Conclusões e Avaliações

o   Recomendações

o   Resumo

 

Estas seções foram criadas buscando apresentar uma forma sistemática para abordar problemas encontrados nos estudos de saúde e medicina. Cada seção do sistema permite ao usuário a documentação das informações obtidas e das ações realizadas em cada fase de estudo do problema.

Como o sistema suporta o acesso de três tipos de usuários, na parte de edição e desenvolvimento de problemas também há diferenciações. Desta forma:

·         Tutores podem editar os campos referentes ao planejamento para desenvolvimento/estudo do tema e também os de avaliação da equipe que ele acompanha;

·         Alunos podem editar os campos relativos ao desenvolvimento propriamente do tema; e

·         Professores podem editar/visualiar todos os campos de trabalho relativos a alunos e tutores.

Além destas questões, o sistema, em casos de problemas já fechados, só disponibiliza os dados para visualização.

Outra convenção da parte de edição e visualização de problemas se dá quanto à primeira edição. Nos casos de campos ainda não editados, o sistema avisa ao usuário de forma destacada, por exemplo, numa descrição de cenário:

 

 

Ao longo deste documento, na maioria dos campos, será convencionado que o problema se encontra aberto e, portanto serão mostradas as telas de edição (seguintes a esta).

Outra distinção se dá na identificação do usuário, pois o mesmo pode ser: professor, tutor ou aluno. Esta identificação se encontra na parte superior da tela, abaixo do menu de navegação:

Depois de passar por essas etapas e as  informações estiverem corretas deve-se clicar em  “Salvar”. Caso contrário deve-se clicar em “Cancelar” e está informação será apagada.

 

 

 

4.4 Seção para desenvolvimento do problema

Nesta seção o estudante deverá descrever as atividades realizadas nos encontros com os tutores.

No decorrer da construção do relato das atividades, comentários podem ser adicionados por todos do grupo, inclusive o tutor. Isso faz com que as discussões possam ser realizadas no próprio ambiente, simultaneamente a construção do relatório. Esses comentários serão visíveis no ambiente do desenvolvimento e não estarão disponíveis no relatório final.

A conclusão e a avaliação dos alunos sobre o aprendizado serão adicionados nesta seção, assim como sugestões de soluções ou continuidade do tema abordado.

 

4.4.1 Desenvolvimento – Relatório de Atividades

O sistema apresenta na seção: “Desenvolvimento – Relatório de Atividades”, o registro de todas as atividades efetivamente realizadas para a resolução do problema. Para editá-lo, caso o problema esteja aberto, será mostrado o botão “Editar”.

Esta tela é exibida selecionando a opção Relatório de Atividades do menu esquerdo.

 

A tela seguinte é conseqüência da escolha de edição na tela principal de Relatório de Atividades.

Os dados informados para Atividades Realizadas podem ser atualizados clicando no botão “Salvar”. Para descartar, clica-se em “Cancelar”

 

4.4.2 Desenvolvimento – Comentário

Nesta seção podem ser adicionados comentários orientadores e em seguida realiza-se atualização clicando-se no botão “Salvar”.

 

 

 

4.4.3 Desenvolvimento – Conclusões e Avaliações

Nesta seção será adicionada a descrição dos resultados obtidos e a avaliação da equipe sobre seu aprendizado em relação ao tema.

Esta tela é exibida selecionando a opção Conclusões e Avaliações do menu esquerdo.

 

Para editá-lo, caso o problema esteja aberto, será mostrado o botão “Editar”.

A tela seguinte será mostrada para que se adicione as informações relativas as conclusões e avaliações da equipe. Para realizar a atualização, clica-se em “Salvar”. Caso deseje descartar, clica-se em “Cancelar”.

 

4.4.4 Desenvolvimento – Recomendações
